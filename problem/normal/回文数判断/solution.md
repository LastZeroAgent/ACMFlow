# 回文数判断 - 题解

## 算法分析

判断回文数最直观的方法是将整数转为字符串，然后检查首尾字符是否对称。但题目要求 ACM 风格，更推荐使用数学方法反转数字的一半（避免反转整个整数可能导致的溢出）。具体思路如下：
1. 所有负数都不是回文数，直接返回 No。
2. 如果数字非负且末尾为 0（但数字本身不为 0），则不是回文数，例如 10、100。
3. 对于剩余情况，我们每次取出 n 的最后一位数字，将其添加到反转数 reversed 的末尾，同时将 n 去掉最后一位，直到 n <= reversed。此时：
   - 若数字长度为奇数（如 12321），反转后的数 reversed 会多出一位，比较 n 与 reversed/10 是否相等。
   - 若数字长度为偶数（如 1221），则比较 n 与 reversed 是否相等。
4. 该方法只反转了数字的一半，避免了溢出，且时间复杂度 O(log₁₀(n))，空间复杂度 O(1)。

**算法选择理由**：整数反转法无需额外空间，效率高；相比于字符串法，更体现对数据结构的理解。

**复杂度分析**：
- 时间复杂度：O(log₁₀(n))，其中 n 是输入整数，每次循环减少一位。
- 空间复杂度：O(1)，只使用了常数个变量。
