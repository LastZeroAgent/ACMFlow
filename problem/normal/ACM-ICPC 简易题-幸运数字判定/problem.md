# ACM-ICPC 简易题-幸运数字判定
 ## 题目描述
给定若干个正整数，请判断每个数字是否为"幸运数字"。幸运数字的定义如下：将数字的各位按顺序分为奇数位和偶数位（从最高位开始，第一位为奇数位），若奇数位上的数字之和等于偶数位上的数字之和，则该数字为幸运数字。

示例说明：
- 数字 `1221` 是幸运数字，因为奇数位(1+2)=3，偶数位(2+1)=3。
- 数字 `1234` 不是幸运数字，因为奇数位(1+3)=4，偶数位(2+4)=6。

## 输入格式
第一行包含一个整数 n (1 ≤ n ≤ 100)，表示需要判断的数字个数。
接下来 n 行，每行包含一个正整数 x (1 ≤ x ≤ 10^18)。

## 输出格式
输出 n 行，每行对应一个输入数字的判断结果。如果是幸运数字，输出 "YES"，否则输出 "NO"。

## 样例

### 样例 1

**输入：**
```
3
```

**输出：**
```
NO
```

### 样例 2

**输入：**
```
1221
```

**输出：**
```
YES
```

## 数据范围
1. 注意数字的位数可能是奇数或偶数，需正确处理最后一位的归属。
2. 大数处理建议使用字符串而非数值类型，避免溢出。
3. 空位或前导零不会出现在输入中。

## [详细的解题思路分析]
本题的核心是将数字转换为字符序列，然后分别累加奇数位和偶数位的数字之和。具体步骤如下：
1. 将输入数字转为字符串，便于逐位访问。
2. 初始化两个累加器 sum_odd 和 sum_even。
3. 遍历字符串的每个字符，索引从 0 开始：
   - 如果索引是偶数（对应实际的第奇数位），加到 sum_odd。
   - 如果索引是奇数（对应实际的第偶数位），加到 sum_even。
4. 比较两个累加器的值，相等则返回 "YES"，否则返回 "NO"。
